# خلفية SSH — russh مع اتصال مشترك

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يحتاج Evernight إلى SSH للقذيفة البعيدة وعمليات الملفات والوصول إلى الطرفية. كان لدى قاعدة الكود مبدئيًا ثلاث تطبيقات منفصلة لمعالجات SSH (FileHandler، SshHandler، TerminalHandler) بمنطق اتصال مكرر. يشترط المشروع Rust نقيًا (لا استدعاءات `Command::new("ssh")` عبر الصدفة).

## القرار

استخدام `russh` (تنفيذ SSH-2 نقي بـ Rust) كخلفية SSH. دمج جميع تطبيقات معالج SSH في `DefaultSshHandler` واحد في `remote/connection.rs` مع دالة `connect_session()` مشتركة. تشارك جميع عمليات SSH (القذيفة، الملفات، الطرفية) تجريد الاتصال هذا.

## النتائج

### إيجابية

- مصدر واحد للحقيقة لمنطق مصادقة SSH
- سهولة إضافة تجميع الاتصالات لاحقًا

### سلبية

- قد يتخلف `russh` عن OpenSSH في الحالات الحدية
- لا يوجد إعادة توجيه وكيل SSH المدمج بعد
