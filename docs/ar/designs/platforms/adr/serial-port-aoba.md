# الاتصال بالمنفذ التسلسلي عبر aoba

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يحتاج Evernight إلى دعم المنفذ التسلسلي لإدارة الأجهزة المضمنة واختبار البروتوكولات الصناعية (Modbus RTU). الكريت الشقيق `aoba` يوفر بالفعل تعداد المنفذ التسلسلي عبر الأنظمة الأساسية، واستخراج VID/PID/serial، ووظيفة Modbus RTU/TCP الرئيسية.

## القرار

تفويض جميع عمليات المنفذ التسلسلي إلى aoba. تعرّف وحدة `serial` في Evernight الأنواع (`SerialConfig`، `SerialPortInfo`) وتنفذ `TerminalBackend` عبر نقل تسلسلي، منادية aoba لإدخال/إخراج المنفذ الفعلي. الكشف التلقائي عن البروتوكول (مسح معدل البود/التكافؤ لـ Modbus RTU) يُفوض أيضًا إلى aoba.

## النتائج

### إيجابية

- يعيد استخدام كود مجرب؛ يتعامل aoba مع الحالات الحدية عبر الأنظمة الأساسية.

### سلبية

- يضيف اعتمادية على aoba؛ تتطلب ميزة التسلسلي توفر aoba.
