# Соглашения логирования CLI

## Обзор

Лог-вывод обёртки CLI shittim-chest следует соглашениям, согласованным с entelecheia, используя экосистему `tracing`, выводя в stderr в компактном человекочитаемом формате.

## Выбор фреймворка

| Компонент | Выбор | Причина |
| --- | --- | --- |
| Фреймворк логирования | `tracing` | Стандарт экосистемы Rust, согласован с entelecheia |
| Подписчик | слой fmt `tracing-subscriber` | Компактный вывод, не требуется разбор JSON |
| Формат времени | `ShortTimer` (ЧЧ:ММ:СС) | Удобен для терминала, согласован с CLI entelecheia |
| Цель вывода | stderr | Отделён от stdout, не мешает пайпам |

## Код инициализации

```rust
use chrono::Local;
use tracing_subscriber::fmt::time::FormatTime;

struct ShortTimer;

impl FormatTime for ShortTimer {
    fn format_time(&self, w: &mut tracing_subscriber::fmt::format::Writer<'_>) -> std::fmt::Result {
        let now = Local::now();
        write!(w, "{} ", now.format("%H:%M:%S"))
    }
}

// Инициализация
tracing_subscriber::fmt()
    .with_env_filter(EnvFilter::new(&args.log_level))
    .with_target(false)          // скрыть пути модулей
    .with_timer(ShortTimer)      // формат ЧЧ:ММ:СС
    .compact()                   // компактный режим
    .with_writer(std::io::stderr) // вывод в stderr
    .init();
```

## Сравнение форматов

| Режим | Пример вывода | Сценарий использования |
| --- | --- | --- |
| CLI (текущий) | `14:23:05  INFO создание сети shittim-chest...` | Разработка, эксплуатация |
| Сервер (будущий) | `{"timestamp":"...","level":"INFO","message":"..."}` | Сбор логов в продакшене |

## Параметр --log-level

CLI принимает параметр `--log-level` / `-l` (по умолчанию `info`):

```text
shittim-chest --log-level debug dev
shittim-chest -l trace status
```

Поддерживаемые уровни: `trace`, `debug`, `info`, `warn`, `error`.

## Соглашения использования уровней логов

| Уровень | Назначение | Типичные сценарии CLI |
| --- | --- | --- |
| `info` | Важные операции | Создание/запуск/остановка контейнера, начало/завершение миграции |
| `warn` | Потенциальные проблемы | Повторные попытки миграции, контейнер существует, но в аномальном состоянии |
| `error` | Ошибки | Падение контейнера, сбой миграции, сбой создания сети |
| `debug` | Отладочная информация | (В настоящее время не используется, зарезервировано) |
| `trace` | Детальный поток | (В настоящее время не используется, зарезервировано) |

## Принципы дизайна

1. **CLI не проглатывает ошибки**: Все ошибки распространяются вверх через `anyhow::Result`; `main()` автоматически печатает цепочку ошибок.
1. **Начало каждой операции логируется**: `создание сети...`, `выполнение миграций...`, `сборка shittim_chest...` — пользователь знает, что делает CLI.
1. **Завершение каждой операции подтверждается**: `shittim-chest запущен на 0.0.0.0:80`, `все сервисы запущены`.
1. **Тихо успешные операции не логируются**: `ensure_network` не печатает, если сеть уже существует, чтобы избежать шума.
1. **Логи контейнера извлекаются через Docker API**: CLI сам не пишет бизнес-логи, только логи операций оркестрации.
