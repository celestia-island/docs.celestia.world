# وثائق التصميم

> **مهم**: هذه وثائق تصميمية طموحية. تصف البنية المعمارية ومجموعة الميزات المقصودة — وليس حالة التنفيذ الحالية.
> للحصول على تقييم صادق لما هو منفّذ فعليًا، راجع
> [ARCHITECTURE.md](../../designs/core/architecture.md) في جذر المستودع، وتحديدًا
> جدول **فحص واقع المكوّنات (Component Reality Check)** وقسم **الفجوات الحالية (Current Gaps)**.
> 2 من أصل 12+ وكيل مجال Layer-2 المخطط لها نشطة في مساحة العمل
> (أتمتة الويب؛ الهندسة البرمجية الكلاسيكية — التحليل الساكن، مراجعة الكود،
> مقاييس الجودة، إعادة الهيكلة، LSP). التصميمات المتبقية محفوظة للتنفيذ المستقبلي.
