# 보안 정책

## 취약점 신고

**보안 취약점에 대해 공개 이슈를 열지 마십시오.**

[GitHub Security Advisories](https://github.com/celestia-island/arona/security/advisories/new)를 통해
비공개로 신고하십시오. GitHub Security Advisories를 이용할 수 없는 경우,
security@celestia.world로 명확한 설명과 재현 단계를 포함하여 메인테이너에게 이메일을 보내십시오.

## 범위

포함 대상:

- 인증 우회, JWT/OAuth 취약점, 세션 처리 결함
- API 키/자격 증명 노출 또는 부적절한 저장
- 권한 부여 및 RBAC 집행 공백
- 주입 취약점 (SQL, 명령어, SSRF, XSS)
- 안전하지 않은 역직렬화, 경로 탐색, SSRF
- 권한 상승 또는 교차 테넌트 접근을 허용하는 이슈

제외 대상:

- 본 프로젝트를 통해 악용할 수 없는 상위 의존성의 취약점
- 문서화된 지침에 반하는 안전하지 않은 구성의 자체 호스팅 배포
- 공개 LLM 제공자 엔드포인트에 대한 서비스 거부

## 대응

| 단계 | 목표 |
| --- | --- |
| 에이전트 확인 응답 | 10분 |
| 담당자 확인 응답 | 1 영업일 |
| 초기 평가 | 3 영업일 |
| 수정 또는 완화 조치 | 30 영업일 (심각도에 따라 다름) |

다음 사항을 포함해 주십시오: (1) 영향을 받는 구성 요소 및 버전, (2) 공격 경로 및 영향,
(3) 재현 단계, (4) 제안된 완화 조치.

## 지원 버전

`main` / `dev` 브랜치의 최신 릴리스 라인만 보안 수정을 받습니다.
