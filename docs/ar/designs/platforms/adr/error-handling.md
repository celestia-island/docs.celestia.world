# معالجة الأخطاء — thiserror مع Result الخاص بالكريت

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يحتاج Evernight إلى نوع خطأ موحد عبر جميع الوحدات (الشاشة، SSH، الأجهزة، الشبكات، الأنفاق). يجب أن تكشف المكتبة واجهة برمجية نظيفة للأخطاء مع إبقاء معالجة الأخطاء الداخلية مريحة.

## القرار

استخدام `thiserror` لاشتقاق تعداد `EvernightError` مع تطبيقات عرض `#[error(...)]` خاصة بكل متغير. تعريف `pub type Result<T> = std::result::Result<T, EvernightError` كنوع النتيجة على مستوى الكريت. يلتقط كل متغير سياقًا خاصًا بالنطاق (ScreenCapture، Ssh، Tunnel، إلخ) كـ String بدلًا من تغليف أنواع الأخطاء الخارجية، لتجنب تسريب تفاصيل الاعتماديات الداخلية.

## النتائج

### إيجابية

- واجهة برمجية مستقرة للأخطاء العامة؛ يطابق المستهلكون متغيرات التعداد دون معرفة التفاصيل الداخلية

### سلبية

- بعض فقدان المعلومات من تحويل `String`
- لا يوجد تسلسل أخطاء منظم
