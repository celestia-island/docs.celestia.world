# حول Shittim Chest (什亭之匣)

## الإصدار 0.1.0

Shittim Chest هو الغلاف الموجّه للمستخدم لمنصة [entelecheia](https://github.com/celestia-island/entelecheia) للتعاون متعدد الوكلاء، مبنية بـ Rust و Vue 3.

## الهندسة المعمارية

يتكون Shittim Chest من عدة مكونات تعمل معًا لتوفير تجربة مستخدم كاملة:

- **arona** — واجهة الدردشة التي تستخدمها حاليًا، تتميز بالاستجابات المتدفقة، توليد الصور، مراقبة حالة الوكيل، نافذة التفكير، عارض الأجهزة البعيدة، ودعم متعدد اللغات.
- **`shittim_chest`** — الخلفية الموحدة Rust + Axum التي تتعامل مع المصادقة (JWT + OAuth)، توجيه LLM المستقل، واجهة برمجة تطبيقات الدردشة، توليد الصور، دخول webhook، وكيل scepter، وإشارات الأجهزة البعيدة.

## العلاقة مع Entelecheia

[entelecheia](https://github.com/celestia-island/entelecheia) هو محرك التنسيق متعدد الوكلاء الأساسي. يوفر زمن تشغيل الوكيل (scepter، 13 وكيل متخصص، زمن تشغيل Cosmos/IEPL). يتعامل Shittim Chest مع كل ما يتفاعل معه المستخدم مباشرة — الهوية، العرض، والتواصل.

المشروعان مفصولان بالتصميم: يدير entelecheia تنسيق الوكلاء، بينما يدير shittim-chest هوية المستخدم والعرض. يتواصلان عبر HTTP/WebSocket بمصادقة JWT. تعيش بيانات اعتماد تسجيل الدخول في `shittim_chest_db`؛ تعيش بيانات الصلاحيات والهوية في entelecheia_db. يسمح هذا الفصل للغلاف الأمامي بالتطور بشكل مستقل عن نواة الوكيل.

## العلاقة مع Hikari

[hikari](https://github.com/celestia-island/hikari) هي بوابة العبور وطبقة التوجيه لمنظومة Celestia Island. تعمل كنقطة الدخول لكل حركة المرور الخارجية، حيث تتعامل مع توجيه الطلبات، موازنة الأحمال، ووظيفة بوابة API بين shittim-chest و entelecheia والخدمات الأخرى.

## العلاقة مع Tairitsu

[tairitsu](https://github.com/celestia-island/tairitsu) هو إطار التطبيق الأصلي عبر المنصات لمنظومة Celestia Island. يوفر عملاء سطح المكتب والجوال المبنيين على Tauri التي تغلف arona كتطبيق أصلي، إلى جانب البنية التحتية للأتمتة والاختبار عبر المتصفح التي تشغل سير عمل التطوير.
