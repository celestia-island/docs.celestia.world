# セキュリティポリシー

## 脆弱性の報告

**セキュリティ脆弱性について公開Issueを作成しないでください。**

脆弱性は[GitHub Security Advisories](https://github.com/celestia-island/arona/security/advisories/new)を通じて非公開で報告してください。GitHub Security Advisoriesがご利用いただけない場合は、security@celestia.worldまで明確な説明と再現手順を添えてメンテナーにメールでご連絡ください。

## 対象範囲

対象内:

- 認証バイパス、JWT/OAuthの弱点、セッション処理の欠陥
- APIキー/認証情報の開示または不適切な保存
- 認可およびRBACの強制に関する欠陥
- インジェクション脆弱性（SQL、コマンド、SSRF、XSS）
- 安全でないデシリアライゼーション、パストラバーサル、SSRF
- 権限昇格やクロステナントアクセスを可能にする問題

対象外:

- 本プロジェクトを通じて悪用不可能な上流依存関係の脆弱性
- 文書化されたガイダンスに反する安全でない設定での自己ホスト型デプロイメント
- 公開LLMプロバイダーエンドポイントに対するサービス拒否

## 対応

| 段階 | 目標 |
| --- | --- |
| エージェントによる確認応答 | 10分 |
| 担当者による確認応答 | 1暦日 |
| 初期評価 | 3暦日 |
| 修正または緩和策 | 30暦日（重大度に依存） |

報告には以下を含めてください：(1) 影響を受けるコンポーネントとバージョン、(2) 攻撃ベクトルと影響、(3) 再現手順、(4) 推奨される緩和策。

## サポートバージョン

`main` / `dev`ブランチの最新リリースラインのみがセキュリティ修正を受けます。
