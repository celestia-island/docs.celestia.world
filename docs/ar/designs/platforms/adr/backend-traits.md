# تجريدات السمات TerminalBackend / ViewportBackend / FileBackend

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يحتاج Evernight إلى واجهات خلفية متعددة الأشكال حتى تتمكن الواجهات الأمامية (CLI، TUI، GUI) من استهلاك أي بروتوكول بشكل موحد. بدون سمات مشتركة، ستحتاج كل واجهة أمامية إلى مسارات كود خاصة بكل بروتوكول للطرفية، والعرض الرسومي، وعمليات الملفات.

## القرار

تعريف ثلاث سمات غير متزامنة آمنة للكائنات في جذر الكريت (متاحة دائمًا، بدون ميزة feature flag):

- **`TerminalBackend`** — `read` / `write` / `resize` / `close`
- **`ViewportBackend`** — `render` / `input` / `clipboard` / `close`
- **`FileBackend`** — `list` / `stat` / `get` / `put` / `rm` / `mkdir` / `rename`

تنفذ كل خلفية بروتوكول السمات ذات الصلة. تستهلك الواجهات الأمامية `Box<dyn TerminalBackend` وغيرها.

## النتائج

### إيجابية

- الواجهات الأمامية مستقلة عن البروتوكول؛ يمكن إدراج خلفيات جديدة (مثل RDP) دون تغييرات في الواجهة الأمامية.

### سلبية

- تتطلب كائنات السمات غير المتزامنة عبئًا إضافيًا في `Box::pin`.
- يجب أن يكون تصميم السمة مستقرًا لأن تغييره يكسر جميع المنفذين.
