# مخطط URI لإدخالات الاتصال

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يحتاج فهرس الاتصالات إلى طريقة موحدة لتمثيل اتصالات البروتوكولات المختلفة (SSH، VNC، RDP، التسلسلي، Docker). يوفر مخطط URI واصفات قابلة للقراءة البشرية وقابلة للتسلسل.

## القرار

استخدام مخططات URI خاصة بكل بروتوكول:

- `ssh://user@host:port`
- `vnc://host:5900`
- `rdp://host:3389`
- `serial:///dev/ttyUSB0?baud=9600`
- `docker:///var/run/docker.sock?container=name`

يقوم `ConnectionEntry` بتحليل URI إلى هياكل مكتوبة تحتوي على المخطط والمضيف والمنفذ واسم المستخدم والمسار ومعاملات الاستعلام. الفهرس عبارة عن شجرة من عقد `ConnectionCategory` تحمل الإدخالات.

## النتائج

### إيجابية

- صيغة URI مألوفة؛ قابلة للتسلسل بسهولة؛ تدعم مشاركة الاتصالات عبر النسخ واللصق.

### سلبية

- بعض البروتوكولات لا تتطابق بشكل نظيف مع URI (مثل سياق Kubernetes).
- معاملات سلسلة الاستعلام غير منظمة.
