
# 基本概念

> 現在のコード実態に基づく概念説明

## 概要

Entelecheia（玄枢）は、小規模なモデル可視ツール面、共有ランタイム、複数のクライアントエントリポイントを備えたマルチエージェントプラットフォームです。リポジトリには現在の実装、実験的な機能、設計ドキュメントが混在しているため、本ガイドでは現在のコードで実際にアクティブなコア概念のみを説明します。

## コア概念

### Agent

Agent はプロンプト、skill、MCP ツールを備えたランタイムロールです。

- Layer1 は現在のプラットフォームのコア能力です。
- 現在の workspace でアクティブな組み込み Layer2 は Web Automation です。
- Layer3（設計段階）は `.amphoreus/` ディレクトリから読み込む計画です — 未実装。

### Exec-Only ツール面

モデルはすべての MCP ツールを直接見ることはできません。現在の主なモデル可視ツールは：

- `exec`
- `write_to_var`
- `write_to_var_json`

ランタイム内部では、`exec` 内のコードが ES モジュールインポートを通じてツール関数を呼び出すことができます（例：`import { tool } from 'agent'`）。

### MCP ツール

MCP ツールは内部の構造化された能力インターフェースです。

- 一部は実際に実装されています。
- 一部は部分実装です。
- 一部は依然としてスタブまたはパラメータ検証スケルトンです。

したがって、ドキュメントに記載されているすべてのツールを、安定して提供可能なものとしてデフォルトで理解すべきではありません。

### Skill

Skill はプロンプトベースで定義されたワークフローで、関連ツールを参照し、場合によっては他の skill も参照します。

- 実際のワークフローを駆動できる skill もあります。
- SOP ドキュメントに近く、完全な自動化チェーンではない skill もあります。

### 階層

| 階層 | 現在の意味 |
| --- | --- |
| Layer1 | workspace でコンパイル有効化されたコア Agent |
| Layer2 | Web Automation というアクティブな組み込みドメイン Agent、およびいくつかのアーカイブ設計 |
| Layer3 | ユーザー定義 Agent（計画中、未実装） |

## クライアント

### TUI

現在最も完全で成熟したユーザーエントリポイントは TUI です。

### WebUI

Web UI（arona チャット）と管理パネル（malkuth）は姉妹リポジトリ [shittim-chest](https://github.com/celestia-island/shittim-chest) に移行され、本コードベースから削除されました。本リポジトリの優先インターフェースは TUI です。

### CLI

CLI は存在しますが、一部のコマンドは依然としてプレースホルダー出力です。

### Tauri クライアント

デスクトップ版とモバイル版のコードは既に [shittim-chest](https://github.com/celestia-island/shittim-chest) 兄弟リポジトリに存在しますが、初期統合と見なすのが適切です。IDE 統合（VS Code、IntelliJ）も同様に shittim-chest にあります。

## セキュリティモデルの保守的な表現

- JWT と API キー認証機能が既に存在します。
- 既知の HTTP、WebSocket、MCP パスに対する RBAC マッピングが既に存在します。
- 暗号化された provider キーストレージ機能が既に存在します。
- コンテナハードニングと監査の完全性は依然として不完全です。

特定のコードパスを確認していない限り、双方向 TLS、完全な capability token、全チェーンにわたる厳格なポリシー適用を現在の事実として扱わないでください。
