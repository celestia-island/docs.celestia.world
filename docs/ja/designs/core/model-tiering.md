# モデル階層化システム設計

## 概要

モデル階層化システムはタスクの複雑さに基づいて適切なモデル階層を割り当てるインテリジェントなモデル選択機構であり、品質を確保しつつリソース活用を最大化する。

> **関連ドキュメント**: 本書で定義される3階層モデルシステムは、[自己進化ループシステム](04-self-evolution-loop.md)の基盤である。

## コア原則

### 3階層モデルシステム

```mermaid
graph TB
    subgraph モデル階層
        T1[T1 深層思考<br/>複雑な推論]
        T2[T2 通常思考<br/>標準タスク]
        T3[T3 基本思考<br/>アトミック操作]
    end

    T1 --> |劣化| T2
    T2 --> |劣化| T3
    T3 --> |ファインチューニング進化| T3_Fine[ファインチューニング済みT3]
```

### 階層比較

| 階層 | 位置付け | コスト | 典型的なシナリオ |
| --- | --- | --- | --- |
| T1（深層） | 複雑な推論、判断 | 最高 | アーキテクチャ設計、問題分析 |
| T2（通常） | 標準タスク | 中 | コード作成、ドキュメント生成 |
| T3（基本） | アトミック操作 | 最低 | ファイル読み取り、フォーマット変換 |

## モデル選択機構

### 選択プロセス

```mermaid
flowchart TD
    Request[タスクリクエスト] --> Parse[levelフィールド解析]
    Parse --> Filter{一致する階層モデルをフィルタ}
    Filter --> |利用可能モデルあり| Check{クォータチェック}
    Filter --> |利用可能モデルなし| Downgrade[劣化を試行]
    Check --> |クォータ十分| Select[最高優先度を選択]
    Check --> |クォータ枯渇| Next[次を試行]
    Select --> Execute[タスク実行]
    Downgrade --> Filter
    Next --> Check
```

### 劣化戦略

```mermaid
stateDiagram-v2
    [*] --> Deep: deepタスク
    Deep --> Normal: deepモデル利用不可
    Normal --> Basic: normalモデル利用不可
    Basic --> [*]: 実行またはエラー

    Deep --> [*]: 実行成功
    Normal --> [*]: 実行成功
```

## 設定機構

### スキル/MCP階層アノテーション

各スキルとMCPツールは `level` フィールドを通じて必要なモデル階層を宣言する：

```mermaid
graph LR
    subgraph 設定レイヤー
        S[スキル設定]
        M[MCP設定]
    end

    subgraph Levelフィールド
        L[level: deep/normal/basic]
    end

    S --> L
    M --> L
    L --> |実行時| Select[モデルセレクタ]
```

### 優先度制御

```mermaid
graph LR
    subgraph 優先度要因
        A[ユーザー設定優先度]
        B[モデル階層マッチ]
        C[クォータ状態]
    end

    A --> |最高重み| Sort[ソート]
    B --> |第2重み| Sort
    C --> |フィルタ条件| Filter[フィルタ]
    Filter --> Sort
    Sort --> Select[モデル選択]
```

## 設計上の考慮事項

### コスト最適化

- 低階層モデルを優先
- 自動劣化によりタスク失敗を回避
- クォータ監視アラート

### 品質保証

- 複雑なタスクには高階層必須
- 劣化には実現可能性検証が必要
- 失敗時の自動リトライ

### 拡張性

- カスタム階層のサポート
- 柔軟な優先度設定
- プラグ可能な選択戦略
