# Основные концепции

> Описание концепций на основе текущей реальности кода

## Обзор

Entelecheia (玄枢) — это мультиагентная платформа с небольшой поверхностью инструментов, видимых модели, общей средой выполнения и несколькими клиентскими точками входа. Поскольку репозиторий одновременно содержит текущие реализации, экспериментальные возможности и проектную документацию, это руководство объясняет только те основные концепции, которые уже активны в текущем коде.

## Основные концепции

### Agent

Агент — это роль времени выполнения с промптом, навыками (skills) и MCP-инструментами.

- Layer1 является основой текущих возможностей платформы.
- Единственный активный встроенный агент Layer2 в текущем workspace — Web Automation.
- Layer3 (стадия проектирования) планируется загружать из каталога `.amphoreus/` — ещё не реализован.

### Поверхность инструментов Exec-Only

Модель не видит напрямую все MCP-инструменты. Основные инструменты, видимые модели в настоящее время:

- `exec`
- `write_to_var`
- `write_to_var_json`

Во время выполнения код в `exec` может вызывать функции инструментов через импорт модулей ES (например, `import { tool } from 'agent'`).

### MCP-инструменты

MCP-инструменты — это внутренние структурированные интерфейсы возможностей.

- Некоторые уже полностью реализованы.
- Некоторые реализованы частично.
- Некоторые всё ещё являются заглушками или каркасами проверки параметров.

Поэтому не следует по умолчанию считать каждый инструмент, упомянутый в документации, стабильно работающим.

### Skill (Навык)

Навык — это рабочий процесс на основе промпта, который ссылается на соответствующие инструменты и иногда на другие навыки.

- Некоторые навыки уже могут управлять реальными рабочими процессами.
- Некоторые навыки ближе к документам SOP, чем к полностью автоматизированным цепочкам.

### Уровни

| Уровень | Текущее значение |
| --- | --- |
| Layer1 | Основные агенты, скомпилированные и включённые в workspace |
| Layer2 | Web Automation — активный встроенный доменный агент, плюс несколько архивных проектов |
| Layer3 | Пользовательские агенты (запланированы, ещё не реализованы) |

## Клиенты

### TUI

В настоящее время наиболее полной и зрелой точкой входа для пользователя является TUI.

### WebUI

Web UI (чат arona) и панель управления (plana) были перенесены в сестринский репозиторий [shittim-chest](https://github.com/celestia-island/shittim-chest) и удалены из этой кодовой базы; предпочтительным интерфейсом этого репозитория является TUI.

### CLI

CLI существует, но некоторые команды всё ещё являются заполнителями.

### Клиент Tauri

Код для десктопа и мобильных устройств уже существует в сестринском репозитории [shittim-chest](https://github.com/celestia-island/shittim-chest), но его лучше рассматривать как раннюю интеграцию. Интеграция с IDE (VS Code, IntelliJ) также находится в shittim-chest.

## Консервативное описание модели безопасности

- Возможности аутентификации JWT и API key уже существуют.
- RBAC-отображение для известных путей HTTP, WebSocket, MCP уже существует.
- Возможность зашифрованного хранения ключей провайдеров уже существует.
- Усиление контейнеров и полнота аудита всё ещё не завершены.

Если вы не проверили конкретные пути кода, не считайте двусторонний TLS, полные capability-токены или строгое соблюдение политик на всём пути текущим фактом.
