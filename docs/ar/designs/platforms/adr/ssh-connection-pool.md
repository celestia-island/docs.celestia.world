# مجموعة اتصالات SSH

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

كانت قاعدة الكود الأصلية تفتح اتصال SSH جديدًا لكل عملية (سرد الملفات، تنفيذ أمر). هذا مكلف (TCP + تبادل المفاتيح + مصادقة لكل استدعاء) ولا يتسع للاستخدام التفاعلي مع عمليات متزامنة متعددة.

## القرار

تنفيذ `SshConnectionPool` مفهرس بـ `(host, port, username)`. يتم تأسيس الاتصالات بكسل عند أول طلب وإعادة استخدامها عبر العمليات. يلتف `PooledSshClient` حول `Arc<Mutex<SshSession` مشترك. تزيل فحوصات الصحة الدورية الاتصالات الميتة.

## النتائج

### إيجابية

- تخفيض كبير في زمن الوصول للعمليات المتكررة.
- يمكّن جلسات قذيفة + ملفات + طرفية متزامنة عبر اتصال واحد.

### سلبية

- يجب أن تتعامل المجموعة مع انتهاء الاتصال وإعادة الاتصال.
- قد تُقتل الاتصالات طويلة العمر بواسطة جدران الحماية.
