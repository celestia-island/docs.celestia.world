# وقت التشغيل غير المتزامن — tokio

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يحتاج Evernight إلى وقت تشغيل غير متزامن لإدخال/إخراج الشبكة (SSH، WebRTC، أنفاق TCP)، والعمليات المعتمدة على المؤقتات (حلقات التقاط الإطارات، مهلات البروتوكول)، وإدارة المهام المتزامنة. يقدم نظام Rust البيئي كلًا من tokio و async-std و smol.

## القرار

استخدام `tokio` كوقت التشغيل غير المتزامن الوحيد. إن tokio هو المعيار الواقعي في نظام Rust البيئي غير المتزامن بأغنى دعم للمشغلات. الاعتمادات الرئيسية (russh، webrtc، reqwest) تتطلب tokio بالفعل. استخدام `tokio::runtime::Handle::current()` للإنشاء من سياقات متزامنة.

## النتائج

### إيجابية

- توافق مع النظام البيئي؛ لا حاجة إلى جسور بين أوقات التشغيل

### سلبية

- tokio اعتمادية ثقيلة
- لا يمكن استخدام كريتات async-std أو smol التي تفتقر إلى دعم tokio
