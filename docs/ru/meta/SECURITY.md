# Политика безопасности

## Сообщение об уязвимости

**Не открывайте публичные issues для уязвимостей безопасности.**

Сообщайте о них конфиденциально через
[GitHub Security Advisories](https://github.com/celestia-island/arona/security/advisories/new).
Если GitHub Security Advisories вам недоступны, напишите сопровождающему на
security@celestia.world с чётким описанием и шагами воспроизведения.

## Область действия

В области действия:

- Обход аутентификации, уязвимости JWT/OAuth, недостатки обработки сессий
- Раскрытие или ненадлежащее хранение API-ключей / учётных данных
- Пробелы в обеспечении авторизации и RBAC
- Уязвимости внедрения (SQL, command, SSRF, XSS)
- Небезопасная десериализация, обход пути, SSRF
- Проблемы, позволяющие повышение привилегий или междоменный доступ

Вне области действия:

- Уязвимости в вышестоящих зависимостях, не эксплуатируемые через этот проект
- Самостоятельно развёрнутые инсталляции с небезопасной конфигурацией вопреки документированным рекомендациям
- Отказ в обслуживании публичных конечных точек LLM-провайдеров

## Ответ

| Этап | Цель |
| --- | --- |
| Подтверждение агентом | 10 минут |
| Подтверждение человеком | 1 календарный день |
| Первичная оценка | 3 календарных дня |
| Исправление или смягчение | 30 календарных дней (в зависимости от серьёзности) |

Пожалуйста, укажите: (1) затронутый компонент и версию, (2) вектор атаки
и воздействие, (3) шаги воспроизведения и (4) предлагаемые меры по смягчению.

## Поддерживаемые версии

Только последняя релизная линия в ветках `main` / `dev` получает исправления
безопасности.
