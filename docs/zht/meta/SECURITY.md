# 安全政策

## 回報漏洞

**不要為安全漏洞開立公開 issue。**

請透過
[GitHub 安全諮詢](https://github.com/celestia-island/arona/security/advisories/new)
私下回報。若您無法使用 GitHub 安全諮詢，請發送電子郵件給維護者
security@celestia.world，附上清晰的描述和重現步驟。

## 範圍

範圍內：

- 身分驗證繞過、JWT/OAuth 弱點、會話處理缺陷
- API 金鑰／憑證洩漏或不當儲存
- 授權和 RBAC 強制缺口
- 注入漏洞（SQL、命令、SSRF、XSS）
- 不安全的反序列化、路徑遍歷、SSRF
- 允許權限提升或跨租戶存取的問題

範圍外：

- 上游依賴中無法透過本專案利用的漏洞
- 違反文件指導的不安全設定的自行託管部署
- 針對公開 LLM 提供者端點的阻斷服務攻擊

## 回應

| 階段 | 目標 |
| --- | --- |
| 代理人確認 | 10 分鐘 |
| 人工確認 | 1 個日曆日 |
| 初步評估 | 3 個日曆日 |
| 修復或緩解措施 | 30 個日曆日（取決於嚴重性） |

請包含：(1) 受影響的元件和版本，(2) 攻擊向量和影響，
(3) 重現步驟，以及 (4) 建議的緩解措施。

## 支援的版本

僅 `main` / `dev` 分支上的最新發行版本線接收安全性修復。
