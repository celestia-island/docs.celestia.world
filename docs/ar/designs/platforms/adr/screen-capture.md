# بنية التقاط الشاشة

- **الحالة**: مقبول
- **التاريخ**: 2025-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يتطلب Evernight نظامًا فرعيًا لالتقاط الشاشة عبر الأنظمة الأساسية يغذي الإطارات إلى خط أنابيب بث WebRTC. يجب أن يعمل النظام الفرعي على ويندوز و macOS و Linux بأقل زمن وصول ودعم كل من مسارات الالتقاط المعجلة بالـ GPU والمسارات البرمجية.

القيود الرئيسية:

- **ميزانية زمن الوصول**: يجب أن يبقى الالتقاط إلى التشفير من طرف إلى طرف أقل من 16 مللي ثانية عند 60 إطارًا في الثانية
- **النسخ الصفري حيثما أمكن**: يجب أن تصل الإطارات إلى المشفر دون نسخ غير ضرورية
- **دعم التوصيل الساخن**: قد تظهر الشاشات ووحدات GPU أو تختفي أثناء التشغيل
- **معالجة الأذونات**: يتطلب macOS إذن تسجيل الشاشة؛ يتطلب Linux الوصول إلى بروتوكول XDG/X11 أو Wayland

## القرار

نعتمد **خلفية التقاط مبنية على السمات** مع تطبيقات خاصة بالمنصة يتم اختيارها وقت الترجمة عبر ميزات Cargo. تنفذ كل خلفية السمة `FrameProvider`:

```rust
#[async_trait]
pub trait FrameProvider: Send + Sync {
    async fn enumerate_outputs(&self) -> Result<Vec<OutputInfo>>;
    async fn start_capture(&mut self, output: OutputId, config: CaptureConfig) -> Result<FrameReceiver>;
    async fn stop_capture(&mut self, output: OutputId) -> Result<()>;
}
```

### اختيار الخلفية

| المنصة   | الخلفية الأساسية             | الخلفية الاحتياطية      |
|----------|------------------------------|-------------------------|
| Windows  | DXGI Desktop Duplication     | GDI BitBlt              |
| macOS    | ScreenCaptureKit (SCStream)  | CGWindowListCreateImage |
| Linux    | PipeWire (عبر xdg-desktop-portal) | XShm / XFixes           |

### دورة حياة الإطار

1. يعيد `FrameProvider::start_capture` قناة `FrameReceiver` — قناة MPSC محدودة تحمل هياكل `Frame`
2. يمتلك كل `Frame` مخزنًا مؤقتًا للذاكرة المشتركة (`Arc<FrameBuffer>`) يشير إلى ذاكرة GPU عند توفرها
3. يستهلك مشفر WebRTC من القناة؛ عند إسقاط جميع مراجع `Arc`، يُعاد المخزن المؤقت إلى مجموعة إعادة الاستخدام
4. لا يحظر مؤشر ترابط الالتقاط أبدًا على التشفير — إذا كانت القناة ممتلئة، يتم إسقاط أقدم إطار وزيادة عداد `FrameDropped`

### مساحة اللون والصيغة

- تتفاوض جميع الخلفيات على أعلى صيغة عمق بت متاحة (BGRA8، NV12، أو P010)
- خطوة `ColorSpaceConverter` تتعامل مع التحويل إلى الصيغة المفضلة للمشفر
- يتم الحفاظ على بيانات HDR الوصفية عندما يوفرها المصدر

## النتائج

### إيجابية

- الفصل النظيف بين الالتقاط والتشفير يسمح بالاختبار المستقل
- مسار النسخ الصفري على ويندوز (DXGI) و macOS (ScreenCaptureKit) يبقي زمن الوصول ضمن الميزانية تمامًا
- التصميم المبني على السمات يسمح بخلفيات الطرف الثالث (مثل الشاشات الافتراضية، مصادر الاختبار) دون تعديل الكود الأساسي
- تجميع مخازن الإطارات المؤقتة يقلل ضغط التخصيص في ظل الالتقاط المستمر

### سلبية

- يقدم PipeWire على Linux اعتمادية D-Bus تعقد السيناريوهات بلا واجهة/المضمنة
- إذن تسجيل الشاشة على macOS يتطلب تفاعل المستخدم عند الإطلاق الأول — لا يوجد حل صامت
- صيانة أربع تطبيقات خلفية تزيد سطح الاختبار

### المخاطر والتخفيفات

- **المخاطر**: تغييرات PipeWire API بين التوزيعات. **التخفيف**: التثبيت على واجهة `pw_stream` C API المستقرة وتضمين روابط Rust.
- **المخاطر**: تلعثم DXGI على الحواسيب المحمولة ذات GPU الهجين. **التخفيف**: اكتشاف طوبولوجيا GPU عند الإقلاع وتفضيل GPU المدمج للالتقاط عندما يكون GPU المنفصل يقوم بالعرض.
