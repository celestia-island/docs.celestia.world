# الأساسيات

> شروحات للمفاهيم مبنية على الواقع الحالي للكود

## نظرة عامة

Entelecheia منصة متعددة الوكلاء تستخدم سطح أدوات صغيرًا مرئيًا للنموذج، ووقت تشغيل مشتركًا، ونقاط دخول متعددة للعملاء. نظرًا لأن المستودع يحتوي على التنفيذ الحالي والقدرات التجريبية ووثائق التصميم معًا، فهذا الدليل يشرح المفاهيم الأساسية النشطة بالفعل في الكود الحالي فقط.

## المفاهيم الأساسية

### الوكيل (Agent)

الوكيل هو دور في وقت التشغيل يملك توجيهات ومهارات وأدوات MCP.

- Layer1 هو القدرة الأساسية للمنصة الحالية.
- الـ Layer2 المدمج النشط في مساحة العمل الحالية هو Web Automation.
- Layer3 (مرحلة التصميم) مُخطط ليُحمَّل من الدليل `.amphoreus/` — غير منفذ بعد.

### سطح الأدوات للتنفيذ فقط

لا يرى النموذج جميع أدوات MCP مباشرةً. الأدوات الأساسية المرئية للنموذج حاليًا هي:

- `exec`
- `write_to_var`
- `write_to_var_json`

داخل وقت التشغيل، يمكن للكود الموجود في `exec` استدعاء دوال الأدوات عبر استيراد وحدات ES (مثلاً `import { tool } from 'agent'`).

### أدوات MCP

أدوات MCP هي واجهات قدرات داخلية منظمة.

- بعضها منفذ فعلًا.
- بعضها منفذ جزئيًا.
- بعضها لا يزال هيكلًا وهميًا أو هيكلًا للتحقق من المعاملات.

لذلك، لا ينبغي لك افتراضًا أن تعتبر كل أداة تظهر في الوثائق مستقرة وجاهزة للتسليم بالفعل.

### المهارة (Skill)

المهارة هي سير عمل مُعرَّف بالتوجيهات يشير إلى أدوات ذات صلة وأحيانًا إلى مهارات أخرى.

- بعض المهارات قادرة بالفعل على تشغيل سير عمل حقيقي.
- بعض المهارات أقرب إلى وثائق SOP منه إلى سلاسل أتمتة كاملة.

### المستويات (Tiers)

| المستوى | المعنى الحالي |
| --- | --- |
| Layer1 | الوكلاء الأساسية المجمَّعة والمُفعَّلة في مساحة العمل |
| Layer2 | Web Automation، وكيل المجال المدمج النشط، بالإضافة إلى بعض التصاميم المؤرشفة |
| Layer3 | وكلاء يُعرِّفها المستخدم (مُخطط، غير منفذ بعد) |

## العملاء

### TUI

نقطة الدخول الأكثر اكتمالًا ونضجًا للمستخدم حاليًا هي TUI.

### WebUI

تم نقل واجهة الويب (محادثة arona) ولوحة الإدارة (malkuth) إلى المستودع الشقيق [shittim-chest](https://github.com/celestia-island/shittim-chest) وأُزيلت من قاعدة الكود هذه؛ الواجهة المفضلة لهذا المستودع هي TUI.

### CLI

واجهة CLI موجودة، لكن بعض الأوامر لا تزال تُخرج نصًا مؤقتًا.

### عملاء Tauri

يعيش كود سطح المكتب والجوال بالفعل في المستودع الشقيق [shittim-chest](https://github.com/celestia-island/shittim-chest)، لكن يُفضَّل اعتباره تكاملًا مبكرًا. تكاملات IDE (VS Code وIntelliJ) تقع كذلك في shittim-chest.

## بيان محافظ لنموذج الأمان

- قدرات المصادقة عبر JWT ومفتاح API موجودة.
- تعيينات RBAC لمسارات HTTP وWebSocket وMCP المعروفة موجودة.
- قدرة تخزين مفاتيح المزودين مشفرة موجودة.
- تحصين الحاويات وسلامة التدقيق لا يزالان غير مكتملَين.

ما لم تكن قد تحققت من مسار الكود المحدد، لا تعتبر TLS المتبادل، أو رموز القدرات الكاملة، أو إنفاذ السياسات الصارم عبر السلسلة بأكملها حقيقة قائمة حاليًا.
