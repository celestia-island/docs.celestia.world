# عميل وقت تشغيل الحاويات (Docker/Podman)

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

إدارة الحاويات (Docker، Podman) هي حالة استخدام رئيسية لمدير اتصالات شامل. تشمل العمليات سرد الحاويات وتنفيذ الصدفة وعرض السجلات وتحويل المنافذ.

## القرار

استخدام Docker Engine API عبر مقبس Unix (أو الأنبوب المسمى named pipe على ويندوز). يتم دعم واجهة Docker المتوافقة من Podman عبر نفس مسار الكود. تعريف نماذج حاويات مكتوبة (`ContainerInfo`، `ContainerState`، `ContainerPort`). مستقبلًا: تنفيذ `TerminalBackend` فوق Docker exec attach لقذائف حاويات تفاعلية.

## النتائج

### إيجابية

- Docker API موثق جيدًا ومستقر؛ توافق Podman مجاني.

### سلبية

- يتطلب تشغيل Docker/Podman daemon.
- اختلافات في إصدار API بين إصدارات Docker.
