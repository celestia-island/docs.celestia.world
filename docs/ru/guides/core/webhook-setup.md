
# Настройка платформы Webhook

> Описание текущей компоновки webhook и объёма интеграции

## Обзор

Репозиторий уже содержит интеграции webhook для платформ хостинга кода и чат-платформ, но в целом всё ещё находится на переходном этапе и не является полностью унифицированным, зрелым решением.

Текущая структура каталогов одновременно содержит:

- Старые каталоги для отдельных платформ, такие как `plugins/github-webhook/github/`, `gitee/`, `gitlab/`, `telegram/`, `qq/`, `lark/`
- Более новую реализацию на TypeScript: `plugins/github-webhook/ts/`

Пакет TypeScript в настоящее время подключает:

- GitHub
- Gitee
- GitLab
- Feishu / Lark
- QQ
- Discord
- Telegram

## Что уже работает

- Получение событий webhook или бота
- Пересылка событий в Scepter через WebSocket или вспомогательные вызовы HTTP
- Предоставление эндпоинта проверки здоровья `/health` в сервисе TypeScript

## Что пока нельзя гарантировать по умолчанию

- Единое стабильное решение развёртывания для всех платформ
- Полная цепочка навыков, управляемая Issue, для каждой платформы
- Одинаковый уровень зрелости для всех подключённых платформ

## Пакет TypeScript

Расположение: `plugins/github-webhook/ts/`

Запуск в режиме разработки:

```bash
cd plugins/github-webhook/ts
npm install
npm run dev
```

Сборка для продакшена:

```bash
cd plugins/github-webhook/ts
npm run build
npm start
```

## Ключевые переменные окружения

- `PORT`: порт сервиса webhook, по умолчанию `8000`
- `SCEPTER_URL`: адрес пересылки HTTP, по умолчанию `http://localhost:8424`
- `SCEPTER_WS_URL`: адрес пересылки WebSocket, по умолчанию `ws://localhost:8424/ws`

## Рекомендации по использованию

Возможности webhook можно рассматривать как «уже существующие, но с неравномерной зрелостью». Если вы зависите от определённой платформы, сначала проверьте фактическую реализацию соответствующего маршрутизатора или бота в `plugins/github-webhook/`, прежде чем описывать её как стабильно готовую к продакшену.
