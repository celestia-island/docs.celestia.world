# بنية أعلام الميزات

- **الحالة**: مقبول
- **التاريخ**: 2026-06-09
- **المؤلفون**: فريق Evernight الأساسي

## السياق

يفرض الرسم البياني للاعتماديات الأحادي في Evernight على جميع المستهلكين سحب كل اعتمادية (webrtc، russh، screenshots، sysinfo) حتى لو احتاجوا فقط إلى SSH أو قياس الأجهزة عن بُعد. هذا يزيد من أوقات التجميع وأحجام الملفات الثنائية للمستخدمين النهائيين.

## القرار

استخدام أعلام ميزات Cargo لتقسيم الكريت إلى الميزات التالية:

| الميزة       | تتحكم في                                       |
|--------------|------------------------------------------------|
| `screen`     | وحدة التقاط الشاشة + كريت `screenshots`        |
| `webrtc`     | وحدة WebRTC + كريت `webrtc` (تتضمن `screen`)   |
| `remote-ssh` | وحدة معالج SSH + كريت `russh`                  |
| `hardware`   | وحدة قياس الأجهزة + كريت `sysinfo`             |
| `protocol`   | أنواع البروتوكول/الرسائل (بدون اعتماديات ثقيلة)|
| `tunnel`     | وحدة نفق TCP                                   |
| `full`       | جميع الميزات (افتراضي)                         |

يتحكم كل علم في كل من الوحدة واعتمادياتها. ميزة `webrtc` تتضمن `screen` لأن جلسات WebRTC تتطلب التقاط الشاشة.

## النتائج

### إيجابية

- المستهلكون يجمعون فقط ما يحتاجونه
- تقليل وقت التجميع للاستخدام الجزئي

### سلبية

- ينمو مصفوفة أعلام الميزات؛ يجب اختبار كل مجموعة ميزات في CI
